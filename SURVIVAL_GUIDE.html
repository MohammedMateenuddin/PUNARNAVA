🔑 1. Your Secret Keys (If These Break, Nothing Works)

Your app needs passwords to talk to Google's services. They live in one file:

VITE_FIREBASE_APP_ID=1:51229171731:web:b32386bce5b1ec0bcc1fe6
🚨 If .env file is missing or empty: NOTHING works — no login, no scanning, no data. Create it in the project root folder (same level as package.json). Copy the values above.
⚠️ NEVER push .env to GitHub. It's in .gitignore for a reason. If pushed, Google auto-revokes your keys and everything breaks. If this happens: go to console.cloud.google.com, generate new keys, update .env.

⚡ 3. How to Run the Project

cd E-WASTE          ← go to project folder
npm install         ← download all libraries (only needed once, or after adding new ones)
npm run dev         ← start the app locally
                    ← opens at http://localhost:5173 or :5177
If npm install fails: Delete the node_modules folder and package-lock.json, then run npm install again. This fixes 90% of install issues.
If npm run dev shows errors: Read the FIRST error line. It usually says which file and which line number has the problem. Fix that one thing.

❌ 4. "Points / Leaderboard Not Updating"

Check: Open browser Console (F12 → Console tab). Look for red errors mentioning "Firestore" or "permission-denied". This means your Firestore security rules are blocking writes. Quick fix for development: 
// Firebase Console → Firestore → Rules → Replace with:
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /{document=**} {
      allow read, write: if true;  // ⚠️ ONLY for testing!
    }
  }
}

❌ 5. "Blank White Screen"

This means JavaScript crashed. Open Console (F12) → look for the red error. Common causes:
  • Missing import — a file is trying to use something that doesn't exist
  • Typo in file name — React is case-sensitive. Scanner.jsx ≠ scanner.jsx
  • Missing dependency — run npm install

❌ 6. "Deployed Site Not Working (Vercel)"

3 things to check on Vercel:
  1. Environment Variables: Vercel dashboard → Settings → Environment Variables → Add all 7 VITE_ keys
  2. Redeploy after adding vars: Vercel → Deployments → Redeploy
  3. vercel.json exists in root with SPA rewrite rule (already done)

🔗 5. How The Key Files Connect (Map)

When something breaks, you need to know which file talks to which:

User clicks "Continue with Google"
  → Login.jsx calls loginWithGoogle()
    → AuthContext.jsx runs signInWithPopup()
      → firebase.js provides the auth connection
        → .env provides the API keys
          → Google checks credentials → returns user info
            → AuthContext saves user to Firestore database
              → App.jsx sees user is logged in → shows Scanner page

User clicks "Capture" on Scanner
  → Scanner.jsx captures frame from camera
    → Calls scanDeviceWithRetry() in geminiScanner.js
      → geminiScanner.js reads VITE_GEMINI_API_KEY from .env
        → Sends photo to Google Gemini API
          → metalPrices.js fetches live Gold/Copper prices
            → Results calculated and displayed
              → User clicks "Submit"
                → AuthContext.jsx updates Firestore: points+, devices+, co2+
                  → Leaderboard.jsx auto-updates (real-time listener)

🧠 6. The 5 Concepts You Actually Need to Know

1. State = screen updates automatically

const [scanning, setScanning] = useState(false);
// When you call setScanning(true), React redraws the screen
// to show the scanning animation. You don't manually change HTML.

2. useEffect = "do this when page loads"

useEffect(() => {
  startCamera();  // runs once when Scanner page opens
}, []);           // empty [] = only on first load

3. Context = shared data across all pages

// ANY page can do this to get the logged-in user:
const { currentUser } = useAuth();
// currentUser has: .displayName, .email, .totalPoints, .devicesRecycled

4. async/await = "wait for internet response"

const result = await scanDevice(photo);  // waits for Google to respond
// Without "await", you'd get an empty result because the
// code moves on before Google finishes responding

5. Environment Variables = secrets

// In .env file:    VITE_GEMINI_API_KEY=AIzaSy...
// In code:         import.meta.env.VITE_GEMINI_API_KEY
// VITE_ prefix is required. Without it, Vite won't load the variable.
⚠️ The VITE_ prefix rule: Every environment variable MUST start with VITE_ or your code can't read it. GEMINI_KEY=abc won't work. VITE_GEMINI_KEY=abc will work.

If You Want to Add a New Feature With AI

The right way to prompt AI for changes:
  1. Tell AI which file to edit (use the table from section 2)
  2. Tell AI what the file currently does
  3. Tell AI what you want changed
  4. Always say: "Don't change anything else in the file"

Example: "In src/pages/Scanner.jsx, after the scan results are displayed, add a 'Share on WhatsApp' button that sends the device name and scrap value. Don't change anything else."

Emergency: Everything is Broken

Nuclear option — reset everything: 
1. Delete node_modules folder
2. Delete package-lock.json
3. Make sure .env file exists with all 7 keys
4. Run: npm install
5. Run: npm run dev
6. If still broken: git stash (saves your changes)
   Then: git checkout master (goes back to last working version)
